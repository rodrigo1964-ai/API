═══════════════════════════════════════════════════════════════════════════════
   PROBLEMAS FUNDAMENTALES DE LA ARITMÉTICA DE INTERVALOS
═══════════════════════════════════════════════════════════════════════════════

1. DEPENDENCY PROBLEM
   ────────────────────
   (x - 1.5) * (x - 1.5)  con x ∈ [1, 2]

   Aritmética de intervalos:
   [1, 2] - 1.5 = [-0.5, 0.5]
   [-0.5, 0.5] * [-0.5, 0.5] = [-0.25, 0.25]  ← INCORRECTO

   Resultado correcto: [0, 0.25]

   La variable x aparece dos veces, pero el intervalo las trata como
   independientes, generando SOBREESTIMACIÓN.

2. WRAPPING/HULL PROBLEM
   ──────────────────────
   Conjunto solución real (cerca del mínimo):
   S = {(1.5, 0.0), (1.4, 0.01), (1.6, 0.01), (1.3, 0.04), (1.7, 0.04), ...}

   Este conjunto es NO CONVEXO en el espacio (x, f(x)).

   El HULL (caja envolvente) incluye puntos que NO pertenecen a S.
   → FALSOS POSITIVOS

3. NO CIERRE BAJO OPERACIONES
   ────────────────────────────
   Las operaciones de intervalos no son cerradas bajo inversos:

   x² ∈ [1, 4]  →  x ∈ [-2, 2] (teórico)
   pero si sabíamos x ∈ [1, 2], perdimos esa información.

═══════════════════════════════════════════════════════════════════════════════
   VENTAJAS DE LA MALLA DISCRETA
═══════════════════════════════════════════════════════════════════════════════

✓ EXACTITUD           Evaluación exacta en cada punto, sin aproximaciones
✓ NO CONVEXIDAD       Representa conjuntos no convexos exactamente
✓ SIN WRAPPING        No hay sobreestimación por efecto hull
✓ SIN DEPENDENCY      Cada evaluación usa valores concretos
✓ PUNTOS CRÍTICOS     Captura mínimos, máximos (si están en la malla)
✓ OPERACIONES CERRADAS Las operaciones están cerradas en el conjunto discreto
✓ COMPOSABILIDAD      Fácil de componer y propagar restricciones

═══════════════════════════════════════════════════════════════════════════════
   FILOSOFÍA: REALES EN CONJUNTOS DISCRETOS
═══════════════════════════════════════════════════════════════════════════════

CONTINUO (Teórico)                    DISCRETO (Implementado)
──────────────────                    ───────────────────────
x ∈ ℝ (infinitos puntos)             x ∈ {x₁, x₂, ..., xₙ} (n puntos)
Aritmética de intervalos              Evaluación exacta en cada punto
Hull/wrapping/sobreestimación         Sin aproximación, sin falsos positivos
Dependency problems                   Sin dependencias (valores concretos)
Conjuntos convexos (aproximación)     Conjuntos no convexos (exactos)

TRADE-OFF:
  Tiempo: O(n^k) para k variables    vs    O(2^k) intervalos
  Resultado: EXACTO en discretización    vs    APROXIMADO con hull

═══════════════════════════════════════════════════════════════════════════════
   APLICACIÓN: CONSTRAINT PROGRAMMING
═══════════════════════════════════════════════════════════════════════════════

Esta filosofía es FUNDAMENTAL para CP (Constraint Programming):

→ Dominios son CONJUNTOS DISCRETOS de valores posibles
→ Propagación elimina valores que violan restricciones (PRUNING)
→ Búsqueda explora combinaciones válidas (BACKTRACKING)
→ Soluciones son EXACTAS, no aproximadas
→ No hay falsos positivos del hull

Ejemplo con Gecode:
  IntVar x(1, 20);        // x ∈ {1, 2, 3, ..., 20} (discreto!)
  rel(x*x - x == 12);     // Evaluación exacta en cada valor
  // Soluciones: x ∈ {4} (exacto, sin wrapping)

CONCLUSIÓN
═══════════════════════════════════════════════════════════════════════════════

Los REALES se representan como CONJUNTOS DISCRETOS FINITOS (malla) para:

  1. Evitar wrapping/hull de intervalos
  2. Evitar dependency problem
  3. Representar conjuntos NO CONVEXOS exactamente
  4. Capturar puntos críticos (mínimos, máximos)
  5. Garantizar operaciones CERRADAS
  6. Alinear con la filosofía de Constraint Programming

Sacrificamos EFICIENCIA en tiempo (O(n) vs O(1)) por EXACTITUD y COMPLETITUD
en la discretización elegida.

La precisión se controla con:
  - Paso de la malla (0.1, 0.01, etc.)
  - Factor de escala (10, 100, 1000, etc.)
  - Aritmética exacta de enteros internamente
